# /root LHF 準拠ディレクトリ構造設計

作成日: 2026-04-25

## 1. この文書の目的

この文書は、Aionix user space である `/root` 以下のディレクトリ階層を Linux Hierarchy Filesystem (LHF) 規約に完全に準拠させるための詳細設計を定義する。

Aionix では、Agent Runtime、Skill、Package、State、Log、Config をすべて `/root` 以下に配置する。これらを混乱なく、かつ運用しやすく整理するために、標準的な Linux ディレクトリ慣習に従う。

## 2. 基本方針

### 2.1 LHF 準拠の意義

- 既存 Linux 管理者が直感的に理解できる構成
- ツール・スクリプトの互換性確保
- 将来的な Ubuntu 統合の容易性
- 標準的な権限・アクセス制御への対応

### 2.2 適用規則

- `/root` は Aionix user space のルート
- `/root` 直下は標準的な bin, etc, lib, var のみ
- Aionix 固有設定は `/root/etc/aionix` 以下に限定
- ランタイムデータは `/root/var/lib/aionix` 以下に限定
- ログは `/root/var/log/aionix` 以下に限定
- ソケット・lock は `/root/var/run/aionix` 以下に限定

## 4. ディレクトリ用途詳細

### 4.1 /root/bin

**目的:** Aionix ユーザー向け実行可能ファイル

**配置ルール:**
- 直接呼び出せるコマンド（Python script でも良い）
- すべて executable (`chmod +x`)
- 名前は `aionix-*` プレフィックス必須

**例:**
```
aionix              # メインコマンド
aionix-runtime      # Runtime 起動
aionix-agent        # Agent 実行
aionix-skill        # Skill テスト実行
aionix-package      # Package 管理
aionix-task         # Task 操作
```

### 4.2 /root/etc/aionix

**目的:** Aionix 設定・定義ファイル

**配置ルール:**
- すべて YAML 形式推奨（互換性・可読性）
- テキスト編集可能（バイナリ不可）
- 管理者（root または aionix user）のみ書込

**構成:**

#### /root/etc/aionix/aionix.conf
メイン設定ファイル（各種デフォルト値）

```yaml
version: "1.0"
runtime:
  workspace: /root/var/lib/aionix/workspace
  cache_dir: /root/var/lib/aionix/cache
  memory_dir: /root/var/lib/aionix/memory
logging:
  level: INFO
  dir: /root/var/log/aionix
  archive_days: 30
permission:
  default_policy: conservative
  approval_required: true
```

#### /root/etc/aionix/agents/
Agent 定義ファイル（YAML）

```yaml
# architecture-designer.yaml
name: Architecture Designer
role: system-architect
goal: Design Aionix system architecture
permissions:
  - read_file
  - write_file
  - update_file
  - read_doc
skills:
  - markdown-formatter
  - diagram-generator
```

#### /root/etc/aionix/skills/
Skill 定義ファイル（YAML）

```yaml
# markdown-formatter.yaml
name: Markdown Formatter
version: 1.0
input_schema:
  type: string
  description: Markdown text to format
output_schema:
  type: string
  description: Formatted Markdown
side_effects: []
```

#### /root/etc/aionix/packages/
Package manifest（YAML）

```yaml
# document-writer-package.yaml
name: document-writer
version: 1.0.0
agent: document-writer
bundled_skills:
  - markdown-formatter
  - github-skill
permissions:
  - read_file
  - write_file
  - update_file
dependencies: []
```

#### /root/etc/aionix/policies/
ポリシー定義ファイル

```yaml
# permission-policy.yaml
policies:
  - name: file-governance
    scope: file-operations
    rules:
      - path: /root
        allow: agents
        operations: [read, write, update]
      - path: /root/etc/aionix
        allow: admin
        operations: [read, write, delete]
```

#### /root/etc/aionix/systemd/
systemd ユニットファイル

```ini
# aionix-runtime.service
[Unit]
Description=Aionix Runtime
After=network.target

[Service]
Type=simple
ExecStart=/root/bin/aionix-runtime
Restart=on-failure
User=aionix
WorkingDirectory=/root

[Install]
WantedBy=multi-user.target
```

### 4.3 /root/lib/aionix

**目的:** Aionix ライブラリ・コアコンポーネント実装

**配置ルール:**
- Python (初期段階) / Go (後期) で実装
- モジュール化（各ディレクトリ = Python package）
- `__init__.py` で public API を明確化

**構成:**

#### /root/lib/aionix/core/
Core runtime components

```
task-manager.py         # Task 管理
agent-runtime.py        # Agent 実行エンジン
context-manager.py      # Context 提供
policy-engine.py        # ポリシー適用
permission-manager.py   # 権限チェック
skill-dispatcher.py     # Skill 呼び出し
```

#### /root/lib/aionix/agents/
Agent 実装

```
base-agent.py           # Agent 基底クラス
architecture-designer-agent.py
planning-agent.py
[各エージェント実装]
```

#### /root/lib/aionix/skills/
Skill 実装

```
base-skill.py           # Skill 基底クラス
markdown-skill.py
github-skill.py
[各スキル実装]
```

### 4.4 /root/var/lib/aionix

**目的:** ランタイム状態・キャッシュ・ワークスペース

**配置ルール:**
- 書込頻繁
- プロセスが読み書き可能
- バックアップ対象外（再生成可能）

**構成:**

#### /root/var/lib/aionix/packages/
インストール済みパッケージ

```
document-writer/
├── agent.yaml           # Agent 定義
├── skills/
│   ├── markdown-formatter/
│   └── github-skill/
└── manifest.yaml        # Package manifest
```

#### /root/var/lib/aionix/workspace/
作業ディレクトリ

```
current-task/           # 実行中タスク作業領域
artifacts/              # 生成物一時置き
temp/                   # 一時ファイル
```

#### /root/var/lib/aionix/cache/
キャッシュデータ

```
agent-registry.json     # Agent index (再生成可能)
skill-registry.json     # Skill index (再生成可能)
package-index.json      # Package index (再生成可能)
```

#### /root/var/lib/aionix/state/
実行状態（重要）

```
task-queue.json         # 待機中・実行中 Task 一覧
agent-instances.json    # Agent インスタンス情報
locks/                  # lock ファイル（※注）
```

NOTE: この state は運用中の一時状態。サービス再開始時に初期化される。

#### /root/var/lib/aionix/memory/
Agent Memory（長期保存）

```
user-memory/            # ユーザーが指定した記憶
project-memory/         # プロジェクト固有知識
agent-memory/           # 各エージェント個別記憶
```

### 4.5 /root/var/log/aionix

**目的:** ログファイル（監査・トラブルシューティング）

**配置ルール:**
- テキストフォーマット（JSON Lines 推奨）
- ローテーション対象（`logrotate` または独自スクリプト）
- アーカイブ（30 日以上保持）

**ログファイル:**

```
runtime.log             # Core runtime ログ
agents.log              # Agent 実行ログ
skills.log              # Skill 実行ログ
tasks.log               # Task 実行ログ
errors.log              # エラーログ
audit.log               # 監査ログ（変更記録）
archive/
└── runtime-20260425.log  # 日付別アーカイブ
```

### 4.6 /root/var/run/aionix

**目的:** ランタイムアーティファクト（一時）

**配置ルール:**
- 揮発性（サービス停止時に削除可）
- PID ファイル、socket、lock ファイルのみ
- /dev/shm または tmpfs マウント推奨（パフォーマンス）

**構成:**

```
aionix.pid              # Main process ID
runtime.sock            # IPC socket
task.sock               # Task manager socket
locks/
├── file-locks/         # ファイル lock
├── task-locks/         # Task lock
└── resource-locks/     # リソース lock
tmp/                    # 一時ファイル (session)
```

## 5. ファイルシステム操作ルール

### 5.1 ディレクトリ権限

```
/root/bin/              755 (rwxr-xr-x)
/root/etc/aionix/       750 (rwxr-x---)   # 管理者のみ読取
/root/lib/aionix/       755 (rwxr-xr-x)
/root/var/lib/aionix/   755 (rwxr-xr-x)   # 書込は aionix user のみ
/root/var/log/aionix/   755 (rwxr-xr-x)
/root/var/run/aionix/   755 (rwxr-xr-x)
```

### 5.2 ユーザー・グループ

```
Owner: root
Group: aionix (Runtime が属するグループ)

/root/etc/aionix/*      root:aionix
/root/lib/aionix/*      root:aionix
/root/var/lib/aionix/*  aionix:aionix (または root:aionix)
/root/var/log/aionix/*  aionix:aionix
/root/var/run/aionix/*  aionix:aionix
```

### 5.3 ファイル操作方針

**読取:**
- 設定ファイル: Agent / Skill が読取可
- ログ: Admin または authorized user のみ

**書込:**
- 設定ファイル: Admin のみ (Config Management System 経由)
- State ファイル: Runtime のみ
- ログ: Runtime / Logger daemon のみ
- Cache: Runtime のみ (再生成可能なため)

**削除:**
- 状態ファイル: Admin のみ (再スタート時)
- ログアーカイブ: Retention policy に基づき自動削除
- Cache: 再生成なので削除可 (ただし実行中の Agent には影響)

## 7. 初期セットアップ

### 7.1 ディレクトリ初期化スクリプト例

```bash
#!/bin/bash

# Create directories
mkdir -p /root/{bin,etc/aionix,lib/aionix,share/aionix}
mkdir -p /root/var/{lib/aionix,log/aionix,run/aionix}

# Create subdirectories
mkdir -p /root/etc/aionix/{agents,skills,packages,policies,templates,systemd}
mkdir -p /root/var/lib/aionix/{packages,workspace,cache,state,memory,locks}
mkdir -p /root/var/log/aionix/archive
mkdir -p /root/var/run/aionix/{locks,tmp}

# Set permissions
chmod 755 /root/{bin,lib,var}
chmod 750 /root/etc/aionix
chmod 755 /root/var/lib/aionix
chmod 755 /root/var/log/aionix
chmod 755 /root/var/run/aionix

# Create aionix user (if not exists)
useradd -r -s /bin/bash aionix 2>/dev/null || true

# Set ownership
chown -R root:aionix /root/etc/aionix
chown -R aionix:aionix /root/var/lib/aionix
chown -R aionix:aionix /root/var/log/aionix
chown -R aionix:aionix /root/var/run/aionix
```

### 7.2 デプロイ手順

1. ディレクトリ構造を作成 (上記スクリプト実行)
2. /root/etc/aionix/aionix.conf を配置
3. Agent / Skill 定義ファイルを配置
4. /root/lib/aionix にコンポーネント実装をインストール
5. systemd unit ファイルを /etc/systemd/system/ にリンク
6. サービス起動: `systemctl start aionix-runtime`

## 8. このルール運用の方針

### 8.1 ルール違反検出

File Governor Agent が以下を検出・修正:

- `/root` 直下への不正ファイル配置
- `etc/`, `lib/`, `var/` 以外へのファイル配置
- LHF 規約外のディレクトリ作成
- 権限設定の不正

### 8.2 ルール更新

新しいコンポーネント追加時:

1. どの LHF ディレクトリに属するか決定
2. 本文書に追加記載
3. File Governor Agent 設定を更新
4. デプロイ前に Architecture Reviewer が検証

- ツール・スクリプトの統一化
- セキュリティ・権限管理の明確化
